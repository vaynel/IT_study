# BPMN

## 개념

### 정의
BPMN(Business Process Model and Notation, 비즈니스 프로세스 모델 및 표기법)은 비즈니스 프로세스를 표준화된 방식으로 표현하는 다이어그램입니다. OMG(Object Management Group)에서 표준으로 관리하며, 비즈니스 프로세스를 시각적으로 모델링하고 실행 가능한 프로세스로 변환할 수 있습니다.

### 핵심 특징
- **표준화**: 업계 표준으로 널리 인정받음
- **실행 가능**: BPMN 모델을 실행 가능한 워크플로우로 변환 가능
- **다양한 요소**: 이벤트, 액티비티, 게이트웨이, 플로우 등 풍부한 요소 제공
- **비즈니스 중심**: 비즈니스 프로세스에 최적화
- **협업 지원**: 비즈니스 분석가와 개발자 간 소통 도구

### 역사
- **2004년**: BPMN 1.0 발표
- **2011년**: BPMN 2.0 발표 (현재 표준)
- **현재**: OMG에서 지속적으로 관리 및 개선

### 기본 원칙
1. **비즈니스 프로세스 중심**: 비즈니스 프로세스를 시각적으로 표현
2. **표준화된 표기법**: 일관된 기호 사용
3. **실행 가능성**: 모델을 실행 가능한 워크플로우로 변환
4. **다양한 관점**: 비즈니스와 기술 관점 모두 지원

## BPMN 주요 요소

### 1. 이벤트 (Event)
- **정의**: 프로세스에서 발생하는 사건
- **표기**: 원형
- **종류**:
  - 시작 이벤트: 프로세스 시작 (얇은 원)
  - 중간 이벤트: 프로세스 중 발생 (이중 원)
  - 종료 이벤트: 프로세스 종료 (굵은 원)
- **타입**: 메시지, 타이머, 에러, 신호 등

### 2. 액티비티 (Activity)
- **정의**: 수행되는 작업
- **표기**: 둥근 사각형
- **종류**:
  - 태스크: 단일 작업
  - 서브프로세스: 하위 프로세스
  - 루프, 멀티인스턴스 등

### 3. 게이트웨이 (Gateway)
- **정의**: 프로세스 흐름의 분기와 병합
- **표기**: 다이아몬드
- **종류**:
  - 배타적 게이트웨이 (XOR): 하나의 경로만 선택
  - 병렬 게이트웨이 (AND): 모든 경로 병렬 실행
  - 포함 게이트웨이 (OR): 조건에 따라 하나 이상 선택
  - 이벤트 기반 게이트웨이: 이벤트에 따라 분기

### 4. 플로우 (Flow)
- **정의**: 프로세스 요소 간의 연결
- **표기**: 화살표
- **종류**:
  - 시퀀스 플로우: 순차적 흐름
  - 메시지 플로우: 메시지 교환
  - 연관: 데이터 연결

### 5. 스윔레인 (Swimlane)
- **정의**: 프로세스 참여자 구분
- **표기**: 수평/수직 레인
- **종류**:
  - 풀 (Pool): 프로세스 참여자
  - 레인 (Lane): 풀 내부의 역할/부서

### 6. 아티팩트 (Artifact)
- **정의**: 추가 정보 제공
- **종류**:
  - 데이터 객체: 데이터 표현
  - 그룹: 요소 그룹핑
  - 주석: 설명

## 어디에 사용하면 유리한지

### 1. 복잡한 비즈니스 프로세스
- **상황**: 여러 단계와 참여자가 있는 복잡한 프로세스
- **예시**: 
  - 주문 처리 프로세스
  - 승인 워크플로우
  - 고객 온보딩 프로세스
- **유리한 점**:
  - 프로세스 전체 시각화
  - 참여자 역할 명확화
  - 프로세스 개선 기회 발견

### 2. 워크플로우 자동화
- **상황**: 프로세스를 자동화해야 하는 경우
- **예시**: 
  - BPM 시스템 구현
  - 워크플로우 엔진 사용
  - 프로세스 자동화
- **유리한 점**:
  - 실행 가능한 모델 생성
  - 자동화 설계 가이드
  - 프로세스 실행 및 모니터링

### 3. 여러 부서 협업
- **상황**: 여러 부서가 참여하는 프로세스
- **예시**: 
  - 다단계 승인 프로세스
  - 부서 간 협업 프로세스
- **유리한 점**:
  - 스윔레인으로 역할 명확화
  - 책임 소재 명확화
  - 협업 프로세스 시각화

### 4. 규정 준수 요구
- **상황**: 규정 준수 프로세스 문서화 필요
- **예시**: 
  - 금융 규정 준수
  - 의료 규정 준수
  - 보안 규정 준수
- **유리한 점**:
  - 표준화된 문서화
  - 감사 대응
  - 규정 준수 증명

### 5. 프로세스 개선
- **상황**: 현재 프로세스 개선 필요
- **예시**: 
  - 비효율적 프로세스 개선
  - 프로세스 최적화
  - 자동화 기회 발견
- **유리한 점**:
  - 현재 프로세스 시각화
  - 개선점 식별
  - Before/After 비교

## 어떤 경우에는 과한 경우인지

### 1. 간단한 프로세스
- **상황**: 단순한 순차 작업
- **예시**: 
  - 간단한 데이터 입력
  - 단순한 계산
- **과한 이유**:
  - 구조가 단순하여 문서화 불필요
  - 플로우차트로 충분

### 2. 기술적 프로세스
- **상황**: 기술적 구현 프로세스
- **예시**: 
  - 코드 실행 흐름
  - 시스템 내부 프로세스
- **과한 이유**:
  - UML 액티비티 다이어그램이 더 적합
  - 기술적 관점에 부적합

### 3. 일회성 프로세스
- **상황**: 한 번만 실행하는 프로세스
- **예시**: 
  - 일회성 작업
  - 임시 프로세스
- **과한 이유**:
  - 문서화 ROI 낮음
  - 빠른 실행이 우선

### 4. 빠르게 변경되는 프로세스
- **상황**: 프로세스가 자주 변경됨
- **예시**: 
  - 실험적 프로세스
  - 프로토타입 프로세스
- **과한 이유**:
  - 문서 업데이트 비용 높음
  - 변경 속도가 빠름

### 5. BPM 도구 미사용
- **상황**: BPM 시스템을 사용하지 않는 경우
- **예시**: 
  - 수동 프로세스
  - 문서화만 필요한 경우
- **과한 이유**:
  - BPMN의 실행 가능성 활용 불가
  - 플로우차트로 충분

# 워크플로우 다이어그램

## 개념

### 정의
워크플로우 다이어그램은 업무 프로세스의 단계와 흐름을 표현하는 다이어그램입니다. 작업의 순서, 조건, 책임자를 시각화하여 업무 프로세스를 명확히 표현합니다.

### 핵심 특징
- **단계별 표현**: 업무 프로세스의 각 단계 표현
- **책임자 명시**: 각 단계의 책임자 표현
- **조건 분기**: 의사결정 지점 표현
- **흐름 명확화**: 프로세스 흐름 시각화

### 주요 구성 요소

#### 작업 (Task)
- **정의**: 수행되는 작업
- **표기**: 박스
- **예시**: 문서 검토, 승인, 처리

#### 의사결정 (Decision)
- **정의**: 조건 판단 지점
- **표기**: 다이아몬드
- **예시**: 승인 여부, 조건 확인

#### 시작/종료
- **정의**: 프로세스 시작과 종료
- **표기**: 원형
- **예시**: 프로세스 시작, 완료

#### 책임자
- **정의**: 작업 수행자
- **표기**: 스윔레인 또는 라벨
- **예시**: 부서, 역할, 담당자

# Gantt 차트

## 개념

### 정의

### 핵심 특징
- **시간축 표현**: 시간에 따른 작업 진행 표현
- **작업 기간**: 각 작업의 기간 표현
- **의존성**: 작업 간 의존성 표현
- **진행 상황**: 작업 진행 상황 추적

### 역사
- **1910년대**: Henry Gantt가 개발
- **현재**: 프로젝트 관리의 표준 도구

---

## 사용하는 곳

### 1. 프로젝트 일정 관리
- 프로젝트 일정 계획
- 작업 일정 관리
- 마일스톤 관리

### 2. 리소스 계획
- 리소스 할당 계획
- 리소스 사용량 관리
- 리소스 충돌 해결

### 3. 진행 상황 추적
- 작업 진행 상황 추적
- 일정 지연 모니터링
- 일정 조정

### 4. 의사소통
- 이해관계자에게 일정 공유
- 프로젝트 상태 보고
- 일정 협의

- **유리한 점**:

- **계획 vs 실제**: 계획과 실제 비교
- **시간 경과**: 시간에 따른 변화 추적
- **예측**: 완료 예상 시점 예측

- 스프린트 진행 상황 추적
- 작업량 변화 모니터링
- 완료 예측

### 2. 속도 측정
- 팀 속도 측정
- 스프린트 속도 분석

## 결론

비즈니스 프로세스 및 프로젝트 관리 다이어그램은 비즈니스 프로세스를 시각화하고 프로젝트를 관리하는 강력한 도구입니다. BPMN은 복잡한 비즈니스 프로세스를, 워크플로우 다이어그램은 업무 프로세스를, Gantt 차트와 PERT 차트는 프로젝트 일정을, 칸반 보드는 작업 관리를, 피쉬본 다이어그램은 문제 분석을 각각 지원합니다.

각 다이어그램은 고유한 목적과 사용 사례를 가지고 있으며, 프로젝트의 특성과 요구사항에 맞는 적절한 다이어그램을 선택하는 것이 중요합니다. 적절한 상황에서 사용하면 비즈니스 프로세스 이해도와 프로젝트 관리 효율성을 크게 향상시킬 수 있습니다.

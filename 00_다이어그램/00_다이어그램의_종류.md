## UML 다이어그램

### 개요
UML(Unified Modeling Language)은 소프트웨어 시스템을 시각적으로 표현하기 위한 표준 모델링 언어입니다.

### 종류

#### 1. 구조 다이어그램 (Structure Diagrams)
- **클래스 다이어그램 (Class Diagram)**: 시스템의 정적 구조를 클래스와 관계로 표현
- **객체 다이어그램 (Object Diagram)**: 특정 시점의 객체 인스턴스와 관계 표현
- **컴포넌트 다이어그램 (Component Diagram)**: 시스템의 물리적 컴포넌트와 의존성 표현
- **배치 다이어그램 (Deployment Diagram)**: 하드웨어 노드와 소프트웨어 아티팩트 배치 표현
- **패키지 다이어그램 (Package Diagram)**: 패키지와 의존성 표현
- **컴포지트 구조 다이어그램 (Composite Structure Diagram)**: 클래스의 내부 구조 표현
- **프로파일 다이어그램 (Profile Diagram)**: UML 확장 메커니즘 표현

#### 2. 행동 다이어그램 (Behavior Diagrams)
- **유스케이스 다이어그램 (Use Case Diagram)**: 시스템 기능과 액터의 상호작용 표현
- **액티비티 다이어그램 (Activity Diagram)**: 비즈니스 프로세스나 워크플로우 표현
- **상태 다이어그램 (State Diagram)**: 객체의 상태 변화 표현
- **시퀀스 다이어그램 (Sequence Diagram)**: 객체 간 시간 순서 메시지 교환 표현
- **커뮤니케이션 다이어그램 (Communication Diagram)**: 객체 간 관계와 메시지 교환 표현
- **인터랙션 오버뷰 다이어그램 (Interaction Overview Diagram)**: 여러 인터랙션 다이어그램 조합
- **타이밍 다이어그램 (Timing Diagram)**: 시간 제약 조건이 중요한 상호작용 표현

### 사용 목적
- 소프트웨어 설계 문서화
- 시스템 아키텍처 시각화
- 개발팀 간 커뮤니케이션
- 코드 생성 및 역공학

## 플로우차트

### 개요
프로세스나 알고리즘의 흐름을 단계별로 표현하는 다이어그램입니다.

### 주요 요소
- **시작/종료**: 타원형
- **프로세스**: 직사각형
- **의사결정**: 다이아몬드
- **입력/출력**: 평행사변형
- **연결선**: 화살표

### 사용 목적
- 알고리즘 설계
- 비즈니스 프로세스 문서화
- 의사결정 흐름 표현
- 시스템 프로세스 설명

---

## ER 다이어그램

### 개요
Entity-Relationship Diagram으로 데이터베이스의 엔티티와 관계를 표현합니다.

### 주요 요소
- **엔티티 (Entity)**: 사각형
- **속성 (Attribute)**: 타원형
- **관계 (Relationship)**: 마름모
- **카디널리티**: 1:1, 1:N, N:M

### 사용 목적
- 데이터베이스 설계
- 데이터 모델링
- 데이터 구조 시각화
- 관계형 데이터베이스 설계

---

## 네트워크 다이어그램

### 개요
네트워크 토폴로지, 장비, 연결을 시각적으로 표현합니다.

### 종류
- **토폴로지 다이어그램**: 네트워크 구조 표현
- **네트워크 맵**: 실제 네트워크 구성 표현
- **OSI 모델 다이어그램**: 7계층 구조 표현
- **네트워크 아키텍처**: 네트워크 설계 표현

### 사용 목적
- 네트워크 설계
- 장비 배치 계획
- 트러블슈팅
- 문서화

## 아키텍처 다이어그램

### 개요
시스템 아키텍처를 다양한 관점에서 표현합니다.

### 종류
- **시스템 아키텍처**: 전체 시스템 구조
- **소프트웨어 아키텍처**: 소프트웨어 구성 요소
- **하드웨어 아키텍처**: 하드웨어 구성
- **엔터프라이즈 아키텍처**: 기업 전체 IT 구조

### 사용 목적
- 시스템 설계
- 아키텍처 문서화
- 의사소통 도구
- 의사결정 지원

## BPMN

### 개요
Business Process Model and Notation으로 비즈니스 프로세스를 표준화된 방식으로 표현합니다.

### 주요 요소
- **이벤트**: 원형
- **액티비티**: 둥근 사각형
- **게이트웨이**: 다이아몬드
- **플로우**: 화살표
- **스윔레인**: 수영 레인

### 사용 목적
- 비즈니스 프로세스 모델링
- 프로세스 최적화
- 워크플로우 설계
- 프로세스 자동화

- **마일스톤**: 다이아몬드

### 사용 목적
- 프로젝트 일정 관리
- 작업 의존성 표현
- 리소스 계획
- 진행 상황 추적

## 클래스 다이어그램

### 개요
시스템의 정적 구조를 클래스, 속성, 메서드, 관계로 표현하는 UML 다이어그램입니다.

### 주요 요소
- **클래스**: 3구획 박스 (이름, 속성, 메서드)
- **관계**: 상속, 구현, 연관, 의존
- **가시성**: +, -, #, ~
- **다중성**: 1, *, 0..1 등

### 사용 목적
- 객체지향 설계

초기 값에서 최종 값까지의 변화를 단계별로 표현하는 차트입니다.

### 주요 요소
- **시작값**: 첫 막대
- **증가/감소**: 중간 막대
- **최종값**: 마지막 막대
- **누적**: 단계별 합계

### 사용 목적
- 재무 분석
- 수익 분석
- 변화 추적
- 원인 분석

---

## IDEF 다이어그램

### 개요
Integration Definition으로 시스템 모델링을 위한 표준 다이어그램입니다.

### 종류
- **IDEF0**: 기능 모델링
- **IDEF1**: 정보 모델링
- **IDEF1X**: 데이터 모델링
- **IDEF3**: 프로세스 모델링

### 사용 목적
- 시스템 분석
- 프로세스 모델링
- 데이터 모델링
- 엔터프라이즈 아키텍처

## DFD (Data Flow Diagram)

### 개요
시스템 내 데이터의 흐름을 표현하는 다이어그램입니다.

### 레벨
- **컨텍스트 다이어그램**: Level 0
- **상위 레벨**: Level 1
- **상세 레벨**: Level 2+

### 주요 요소
- **프로세스**: 원형
- **데이터 저장소**: 열린 사각형
- **외부 엔티티**: 사각형
- **데이터 흐름**: 화살표

### 사용 목적
- 시스템 분석
- 데이터 흐름 문서화
- 요구사항 분석
- 시스템 설계

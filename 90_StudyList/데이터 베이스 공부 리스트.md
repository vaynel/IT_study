데이터 엔지니어링의 핵심인 DB(데이터베이스)를 빼놓을 수 없죠. 데이터를 단순히 저장하는 것을 넘어, **대규모 데이터를 어떻게 효율적으로 조회하고 관리할 것인가**에 초점을 맞춘 DB 공부 리스트입니다.

개발자로서 깊이 있게 파고들어야 할 주제들을 5가지 카테고리로 정리했습니다.

---

## 1. RDBMS 심화 (Relational DB)

단순 SQL 작성을 넘어 DB의 내부 동작 원리를 이해해야 성능 최적화가 가능합니다.

* **인덱스(Index)의 원리**: B-Tree, B+Tree 자료구조와 복합 인덱스 설계 전략.
* **트랜잭션과 격리 수준(Isolation Level)**: MVCC(Multi-Version Concurrency Control)와 서비스 성격에 따른 격리 수준 선택.
* **실행 계획(Execution Plan) 분석**: 쿼리가 어떻게 실행되는지 분석하고 인덱스 스캔 vs 풀 스캔을 최적화하는 방법.
* **정규화와 역정규화**: 설계 단계에서의 성능 트레이드오프(Trade-off) 결정.

## 2. NoSQL 및 분산 데이터베이스

데이터의 형태와 규모에 따라 RDBMS의 한계를 극복하는 기술들입니다.

* **CAP 이론 & PACELC 이론**: 분산 시스템에서 일관성(Consistency)과 가용성(Availability) 중 무엇을 우선할 것인가.
* **Document DB (MongoDB)**: 유연한 스키마 설계와 샤딩(Sharding) 구조.
* **Key-Value Store (Redis)**: 캐싱 전략, 자료구조 활용(Sorted Set, Hash), 분산 락(Distributed Lock) 구현.
* **Wide-Column Store (Cassandra)**: 대량의 쓰기 작업에 최적화된 분산 아키텍처.

## 3. OLAP 및 분석용 DB (Data Engineering 핵심)

데이터 엔지니어링에서는 '집계'와 '분석' 속도가 가장 중요합니다.

* **Columnar Storage (열 지향 저장소)**: Row 단위가 아닌 Column 단위로 저장하여 분석 쿼리 속도를 높이는 원리.
* **ClickHouse**: 대규모 로그 분석에 최적화된 DB 엔진 (사용자님의 SIP 데이터 분석에 강력 추천).
* **DuckDB**: 로컬 환경이나 데이터 파이프라인 중간 단계에서 가볍고 빠르게 SQL 분석을 수행하는 임베디드 OLAP.
* **데이터 모델링**: Star Schema, Snowflake Schema 등 분석에 최적화된 테이블 구조 설계.

## 4. 특수 목적 DB (Specialized DB)

최근 보안 및 AI 분야에서 필수적으로 쓰이는 DB들입니다.

* **Vector Database (Milvus, Pinecone)**: LLM(거대언어모델)이나 유사도 검색에 사용되는 벡터 데이터 저장 및 검색.
* **Search Engine (Elasticsearch / OpenSearch)**: 대량의 텍스트 데이터 전문 검색(Full-text Search)과 로그 분석(ELK Stack).
* **Time-Series Database (InfluxDB, Prometheus)**: 시계열 데이터(메트릭, 센서 데이터) 저장 및 윈도우 함수 활용.

## 5. DB 운영 및 아키텍처 (Engineering & Ops)

데이터를 안전하고 확장 가능하게 관리하는 기술입니다.

* **Replication (복제)**: Master-Slave 구조를 통한 읽기 부하 분산과 가용성 확보.
* **Sharding (샤딩)**: 데이터를 여러 DB 서버에 수평으로 분할하여 저장하는 기술.
* **CDC (Change Data Capture)**: DB의 변경 사항을 실시간으로 감지하여 데이터 웨어하우스나 다른 서비스로 전달하는 기술 (Debezium 등).
* **Migration**: 무중단 DB 마이그레이션 전략.

---

### 🛠 실전 적용 제안

사용자님은 현재 **보안 로그(SIP) 분석**을 하고 계시니, 다음 흐름으로 공부를 시작해 보시는 건 어떨까요?

1. **우선순위 1**: **ClickHouse** 또는 **Elasticsearch** (대량 로그를 빠르게 검색/집계하기 위해)
2. **우선순위 2**: **CDC** (운영 DB의 데이터를 분석용 DB로 실시간으로 옮기기 위해)
3. **우선순위 3**: **Redis** (자주 조회되는 위협 IP 리스트 등을 메모리에 올려 속도를 높이기 위해)

이 리스트를 이전에 드린 데이터 엔지니어링/시각화 리스트와 합쳐서 **하나의 '공부 로드맵' 노트로 만들어 드릴까요?** 아니면 특정 DB에 대해 더 자세한 설명이 필요하신가요?

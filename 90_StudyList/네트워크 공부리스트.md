현업 개발자로서 기술적 깊이를 더하고 스펙트럼을 넓히기 위해 필요한 공부 리스트를 5개 카테고리로 나누어 정리해 드립니다. 이미 실무를 하고 계시니, 단순 문법보다는 **원리와 설계, 운영** 관점에 초점을 맞췄습니다.

---

## 1. Computer Science (탄탄한 기본기)

기술 트렌드는 변하지만 CS 지식은 변하지 않는 뿌리입니다.

* **자료구조 & 알고리즘**: 복잡도() 분석, 그래프, 동적 계획법(DP) 등 효율적인 코드 작성을 위한 핵심.
* **운영체제(OS)**: 프로세스와 스레드, 컨텍스트 스위칭, 메모리 관리(가상 메모리), 동기화 문제(Deadlock, Race Condition).
* **네트워크**: HTTP/HTTPS 프로토콜의 깊은 이해, TCP/UDP, DNS, CORS, 그리고 최근의 HTTP/3.
* **데이터베이스**: SQL/NoSQL의 차이, 트랜잭션 격리 수준(Isolation Level), 인덱싱 원리, 쿼리 최적화(Execution Plan 분석).

---

## 2. Software Engineering (지속 가능한 개발)

협업과 유지보수가 쉬운 코드를 작성하기 위한 영역입니다.

* **디자인 패턴**: GoF 디자인 패턴(Singleton, Strategy, Observer 등)을 실무 코드에 적용해 보기.
* **테스트 자동화**: Unit Test(JUnit, Jest), Integration Test, TDD(테스트 주도 개발) 방법론.
* **리팩토링 & 클린 코드**: 가독성 좋은 코드 작성법, 기술 부채를 줄이는 코드 구조 개선.
* **프로그래밍 패러다임**: 객체 지향 프로그래밍(OOP)의 원칙(SOLID), 그리고 함수형 프로그래밍(Functional Programming)의 개념.

---

## 3. Infrastructure & DevOps (확장성 있는 시스템)

코드가 돌아가는 환경을 이해하고 직접 제어하는 능력입니다.

* **컨테이너 기술**: Docker의 기본 사용법과 이미지 최적화, Kubernetes(K8s)의 오케스트레이션 개념.
* **CI/CD**: GitHub Actions, Jenkins 등을 활용한 빌드/테스트/배포 자동화 파이프라인 구축.
* **클라우드 서비스(AWS/Azure/GCP)**: 가상 서버(EC2), 서버리스(Lambda), 스토리지(S3), VPC 네트워크 설계.
* **모니터링 & 로깅**: Prometheus, Grafana, ELK Stack 등을 활용한 시스템 장애 추적 및 성능 분석.

---

## 4. Modern Trends & Specialized (미래 경쟁력)

현재 시장에서 요구하는 기술적 변화에 대응하는 능력입니다.

* **마이크로서비스 아키텍처(MSA)**: 서비스 간 통신(gRPC, Kafka/RabbitMQ), 분산 트랜잭션 관리.
* **보안(Security)**: OWASP Top 10(SQL Injection, XSS 등), OAuth 2.0/JWT 인증 체계, 암호화 원리.
* **AI 프롬프트 엔지니어링 & API 활용**: LLM(대규모 언어 모델)을 개발 프로세스에 통합하거나 API를 활용한 서비스 개발.
* **성능 최적화**: 프론트엔드(렌더링 성능, 번들링 최적화) 또는 백엔드(캐싱 전략 - Redis, 동시성 처리).

---

## 5. Soft Skills (시니어 개발자로의 도약)

기술만큼이나 중요한 커뮤니케이션 기술입니다.

* **문서화 능력**: 요구사항 정의서, API 명세서, 기술 설계 문서(Design Doc) 작성법.
* **코드 리뷰**: 비난이 아닌 성장을 돕는 리뷰 피드백 기술.
* **비즈니스 이해도**: 내가 만드는 기능이 사업적으로 어떤 가치를 주는지 파악하는 안목.
